EXHIBIT A
STATEMENT OF WORK – PHASE 1A MVP
A.1 Project Objective
Developer shall deliver a lean, launch-oriented Phase 1A MVP for Think Funding LLC. The objective is to
provide a functional foundation that supports early merchant intake and internal operations without
expanding into later-phase features.
A.2 In-Scope Deliverables (Phase 1A Only)
The in-scope Deliverables for Phase 1A MVP are limited to the following high-level items:
(a) a professional public-facing website presence for Think Funding at or for use with
www.thinkfundinggroup.com, including a dark mode option with the default appearance set to light
mode;
(b) a secure merchant application intake flow sufficient for initial MVP use;
(c) secure document upload capability for merchant-submitted materials;
(d) a basic internal operations/admin interface sufficient to review submissions and core records at an
MVP level;
(e) core Azure-hosted application setup for the MVP environment;
(f) core database setup using Azure SQL for foundational MVP entities and records;
(g) document storage setup using Azure Blob Storage or equivalent Azure storage selected for the MVP
architecture;
(h) Microsoft Entra-based authentication or access controls as applicable to the MVP scope;
(i) baseline n8n workflow/orchestration setup for submission intake, file organization, and basic
notifications/alerts;
(j) basic Microsoft 365 notification integration (such as Outlook and/or Teams notifications) where
required by the MVP scope;
(k) structured handoff of code, configurations, workflows, and Technical Documentation; and
(l) deployment and handoff sufficient for Client to possess and understand the delivered MVP
foundation.
A.3 Technical Documentation Deliverables
Developer shall prepare and save to Client’s designated SharePoint location detailed Technical
Documentation for all in-scope MVP work. Such documentation shall include, at minimum, the following, to
the extent applicable to the delivered solution:
(a) solution overview and architecture summary;
(b) repository/codebase structure summary;
(c) environment/configuration documentation, excluding secret values;
(d) Azure services used and their role in the MVP;
(e) Azure SQL schema/object overview for the MVP;
(f) n8n workflow descriptions, trigger logic, and operational purpose;
(g) deployment notes and update steps;
(h) dependency list and third-party component summary;
(i) troubleshooting/runbook notes; and
(j) known limitations, deferred items, and assumptions.
A.4 Deliverable Standards
All Deliverables shall be delivered in a reasonably organized, readable, and maintainable manner consistent
with MVP-stage software development. Developer shall provide Client with all source code, compiled code,
